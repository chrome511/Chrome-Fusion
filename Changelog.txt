Changelog:

English:
--------------
v2.00.03
- Return to the original setup routine of PHP-Fusion 7.02.07
- PHP 7.2.x support
- Site administrator with "USER_ID == 1" can edit and delete appointed site administrator
- BUGFIX: Site Moderators will now show Moderator Area instead of Admin Area under User-> Moderators "Moderator Password" instead of "Admin Password"
- BUGFIX: in the admin area, in the "Navigation Panel" "Moderation Overview" is now displayed for moderators instead of "Administration Overview"
- BUGFIX: If a moderator has not yet specified a moderator password (admin password), it is now correctly indicated with "You have not set a moderator password ......"

v2.00.02
- Links in the user_info_panel now underlined
- BUGFIX: Moderators can now be given rights for the infusions
- Admins can no longer edit greater or identical user levels (edit each other admins)

v2.00.01
- admins can not edit the admin rights themselves anymore
- BUGFIX: Moderator menu is not displayed in the user_info_panel
- Stay logged in is now displayed in maintenance mode and in the User_info_panel
- Super Moderator now also has the rights for articles, article categories, forum, news, poll and user log to the standard rights
- Super Moderator can no longer appoint a moderator by default
- Moderator now also has the rights for user log and forum to the standard rights

v2.00.00
- PHP 7 with MySQL/PDO support

v1.00.00
- New userlevel (Moderator & Super Moderator) added
- SSL secure for SMTP host
- Siteadminemail (separate e-mail addresses for the site admin (profil) and emails from the system)
